# NewCar

Полноценный веб-сайт автосалона на Laravel с архитектурой MVC, каталогом автомобилей, личным кабинетом и административной панелью.

## Возможности

- Гостевой каталог автомобилей с поиском, фильтрацией и сортировкой
- Сравнение до 3 автомобилей
- Избранное в личном кабинете
- Кредитный калькулятор на карточке автомобиля
- Карточка автомобиля с заявкой и записью на тест-драйв
- Новости и акции
- Форма обратной связи
- Регистрация, авторизация и восстановление пароля
- Личный кабинет с историей заявок и тест-драйвов (статусы на русском)
- Email-уведомления администратору и клиентам
- Административная панель:
  - дашборд со статистикой
  - CRUD автомобилей с загрузкой изображений
  - управление пользователями
  - управление новостями с загрузкой изображений
  - просмотр и обработка заявок, тест-драйвов и сообщений

## Быстрый запуск

1. Установите PHP-зависимости:

```bash
composer install
```

2. Установите Node.js и npm, затем фронтенд-зависимости:

```bash
npm install
```

3. Соберите ассеты:

```bash
npm run build
```

4. Убедитесь, что MySQL запущен, и создайте базу данных `newcar`.

5. Проверьте настройки в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=newcar
DB_USERNAME=root
DB_PASSWORD=

ADMIN_EMAIL=admin@newcar.local
MAIL_MAILER=log
```

6. Выполните миграции, сидирование и создайте симлинк для загрузок:

```bash
php artisan migrate --seed
php artisan storage:link
```

Если база уже была создана ранее, для новых таблиц выполните:

```bash
php artisan migrate
```

7. Запустите проект:

```bash
php artisan serve
```

## Структура

- `app/Models` — модели предметной области
- `app/Enums` — статусы заявок и тест-драйвов
- `app/Http/Controllers` — контроллеры клиентской части, авторизации и админки
- `database/migrations` — миграции таблиц
- `database/seeders` — стартовые данные
- `resources/views` — Blade-шаблоны
- `resources/css/app.css` — адаптивные стили
- `routes/web.php` — маршруты сайта

## Важно

- В проекте используется CSRF-защита Laravel.
- Валидация форм реализована на стороне сервера.
- Письма по умолчанию пишутся в лог (`MAIL_MAILER=log`). Для реальной отправки настройте SMTP.
- Изображения можно загружать с диска или указывать по URL.
- Для полноценной фронтенд-сборки нужен установленный `npm`.
